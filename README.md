# shift_web

JavaのServlet・JSP・DAOを用いて作成した、シフト管理Webアプリケーションです。

労働者はカレンダーから勤務可能な日付と時間を選択してシフトを提出できます。雇用者は指定した日付のシフト情報を検索し、勤務可能な労働者を確認できます。

ユーザー情報や提出されたシフト情報はMySQLに保存されます。

## 機能

**1. ユーザー登録**<br>

メールアドレス、ユーザー名、パスワード、ユーザー種別を入力してアカウントを登録できます。

ユーザー種別は、以下の2種類から選択します。

* 雇用者
* 労働者

入力した情報は確認画面に表示され、内容を確認した後にMySQLへ登録されます。

**2. ログイン**<br>

登録済みのメールアドレスとパスワードを入力してログインできます。

データベースに保存されているユーザー情報と入力内容が一致した場合、ユーザー種別に応じた画面へ移動します。

**3. 労働者によるシフト提出**<br>

労働者はカレンダーから勤務可能な日付を選択し、0時から23時までの時間帯を指定してシフトを提出できます。

1つの日付に対して複数の時間帯を選択できます。

**4. 過去に提出したシフトの表示**<br>

労働者が過去に提出したシフト情報をデータベースから取得し、カレンダー上に表示します。

提出済みのシフトを確認しながら、新しいシフトを追加できます。

**5. 雇用者によるシフト検索**<br>

雇用者は日付を指定して、労働者から提出されたシフト情報を検索できます。

指定した日に勤務可能なユーザーや時間帯を確認できます。

**6. トークンではなくセッションによるユーザー情報の保持**<br>

ログイン後は、メールアドレス、ユーザー名、ユーザー種別などの情報をセッションに保存します。

これにより、ページを移動してもログイン中のユーザー情報を利用できます。

**7. データベースへの情報保存**<br>

ユーザー情報やシフト情報はMySQLに保存されます。

アプリケーションを終了した後も、登録情報や提出済みのシフト情報が保持されます。

## 技術スタック

* フロントエンド：HTML、CSS、JavaScript、JSP
* バックエンド：Java、Servlet
* データベース：MySQL
* データベース接続：JDBC
* Webサーバー：Apache Tomcat
* アーキテクチャ：MVC、DAOパターン

## データベース

データベース名には`shift_sample`を使用しています。

### `enroll`テーブル

ユーザーの登録情報を管理します。

| カラム名       | 内容               |
| ---------- | ---------------- |
| `email`    | メールアドレス          |
| `username` | ユーザー名            |
| `password` | パスワード            |
| `usertype` | 雇用者または労働者のユーザー種別 |

### `shift`テーブル

労働者が提出したシフト情報を管理します。

| カラム名         | 内容                   |
| ------------ | -------------------- |
| `email`      | シフトを提出したユーザーのメールアドレス |
| `username`   | シフトを提出したユーザー名        |
| `shift_date` | 勤務可能な日付              |
| `shift_hour` | 勤務可能な時間              |
| `is_working` | シフトの状態を表す値           |


## 処理の流れ

### ユーザー登録

```text
ユーザー登録画面
    ↓
Shift_confirmServlet
    ↓
確認画面
    ↓
Shift_dbinsertServlet
    ↓
MySQLへユーザー情報を登録
```

### ログイン

```text
ログイン画面
    ↓
Shift_loginconfirm
    ↓
Shift_login_DAO
    ↓
MySQLのユーザー情報と照合
    ↓
労働者画面または雇用者画面
```

### シフト提出

```text
カレンダーから日付と時間を選択
    ↓
Shift_submit
    ↓
Shift_shift_DAO
    ↓
MySQLへシフト情報を登録
    ↓
提出済みシフトを再表示
```

### シフト検索

```text
雇用者が日付を指定
    ↓
Shift_search
    ↓
Shift_DAO
    ↓
指定日のシフト情報を取得
    ↓
雇用者画面に検索結果を表示
```

## 工夫した点

* Servlet、JSP、DAOに役割を分け、MVCを意識した構成にした
* `PreparedStatement`を利用してSQLを実行した
* セッションを使用してログイン中のユーザー情報を保持した
* JavaScriptでカレンダーと時間選択ボタンを動的に生成した
* 1つの日付に対して複数の時間帯をまとめて送信できるようにした
* 過去に提出したシフトをカレンダー上で確認できるようにした
* 雇用者と労働者で利用できる機能と画面を分けた
